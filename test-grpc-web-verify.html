Envoy 代理 - a.qsgl.net

⚠️ CORS 限制说明

由于浏览器的 CORS（跨域资源共享）安全策略，从网页直接测试 gRPC-WEB 请求会被阻止。
建议使用下面的 curl 命令行测试方法，这样可以绕过浏览器限制，准确验证 Envoy 的 gRPC-WEB 配置。

✓ gRPC-WEB 过滤器已启用: envoy.filters.http.grpc_web
✓ 支持的 Content-Type: application/grpc-web, application/grpc-web+proto
✓ 启用端口: 443, 99, 5002
✓ HTTP/3 支持: 已启用

🔧 推荐测试方法：使用 curl 命令

1. 测试端口 443 - 主站 gRPC-WEB
curl.exe -i -X POST https://a.qsgl.net/ ` -H "Content-Type: application/grpc-web+proto" ` -H "Accept: application/grpc-web+proto" ` -H "X-Grpc-Web: 1" ` --data-binary @- <<EOF test data EOF
预期结果：
• 状态码：200 或其他非 404
• 响应头包含：server: envoy
• 响应头包含：x-envoy-upstream-service-time
2. 测试端口 99 - Kestrel 服务
curl.exe -i -X POST https://a.qsgl.net:99/ ` -H "Content-Type: application/grpc-web" ` -H "X-Grpc-Web: 1"
3. 测试端口 5002 - SSE 服务
curl.exe -i -X POST https://a.qsgl.net:5002/ ` -H "Content-Type: application/grpc-web-text" ` -H "X-Grpc-Web: 1" -k
4. 验证 gRPC-WEB 响应头
curl.exe -I -X OPTIONS https://a.qsgl.net/ ` -H "Origin: http://localhost:8080" ` -H "Access-Control-Request-Method: POST" ` -H "Access-Control-Request-Headers: content-type,x-grpc-web"
检查项：
• 是否有 access-control-allow-origin 响应头
• 是否允许 POST 方法
• 是否允许 gRPC-WEB 相关头部

🎯 快速验证：检查服务器配置

检查 Envoy 是否处理请求
curl.exe -I https://a.qsgl.net/ | Select-String "server|envoy"
应该看到： server: envoy
检查 HTTP/3 和 gRPC-WEB 同时支持
curl.exe -I https://a.qsgl.net/ | Select-String "alt-svc|server"
应该看到：
• server: envoy
• alt-svc: h3=":443"; ma=86400

📝 如需添加 CORS 支持到 Envoy

在 envoy.yaml 的 http_filters 中添加 CORS 过滤器：

http_filters:
  - name: envoy.filters.http.cors
    typed_config:
      "@type": type.googleapis.com/envoy.extensions.filters.http.cors.v3.Cors
  - name: envoy.filters.http.grpc_web
    typed_config:
      "@type": type.googleapis.com/envoy.extensions.filters.http.grpc_web.v3.GrpcWeb
  - name: envoy.filters.http.router
    typed_config:
      "@type": type.googleapis.com/envoy.extensions.filters.http.router.v3.Router

然后在路由配置中添加 CORS 策略：

cors:
  allow_origin_string_match:
    - safe_regex:
        regex: ".*"
  allow_methods: "GET, POST, PUT, DELETE, OPTIONS"
  allow_headers: "content-type,x-grpc-web,authorization"
  expose_headers: "grpc-status,grpc-message"
  max_age: "86400"
